ESP32 + Android bridge + Tailscale

Remote RC driving over a phone-mounted bridge.

A brushless RC car controlled from a PC dashboard. The Android phone rides on the car, streams camera/GPS, and bridges Tailscale commands to the ESP32 over the car-side network.

Setup Guide Source Code

Current Status

The project has moved from public-MQTT-only driving to a phone bridge architecture. Public MQTT remains as fallback/status, while primary driving commands are forwarded by the phone using UDP.

Driving Dashboard

Camera-first cockpit with telemetry, map overlay, ESP status, WASD control, and PS4 controller support.

Phone Bridge

Termux script receives Socket.IO commands over Tailscale and sends low-latency UDP packets to the ESP32.

ESP32 Control

UDP command listener, command sequence protection, throttle failsafe, steering servo, and ESC output.

Architecture

Tailscale connects the PC and phone. The phone then bridges commands to the ESP32 locally.

PC
Flask dashboard, WASD, PS4 client
↓ Tailscale / Socket.IO
Android phone on car
phone_bridge.py, IP Webcam, GPSLogger
↓ UDP over hotspot / car-side Wi-Fi
ESP32
Servo steering, ESC throttle, failsafe

Setup

PC

pip install flask flask-socketio python-socketio eventlet paho-mqtt opencv-python pygame
python app.py

Android Phone

pkg update
pkg install python
pip install "python-socketio[client]"
python phone_bridge.py

ESP32

Upload rc_controller/rc_controller.ino. The ESP listens for UDP commands on port 4210.

GPSLogger

http://100.70.113.90:5003/api/location?lat=%LAT&lon=%LON&acc=%ACC&alt=%ALT&speed=%SPD&dir=%DIR

Controls And Robustness

WASD

  • W: brake / reverse
  • S: forward
  • A: left
  • D: right

PS4 Controller

Supported by ps4controller.py. Current hardware issue: Bluetooth pairing is unreliable and USB cable is loose.

Failsafe

The ESP cuts throttle if command packets stop, and ignores stale command sequence numbers.
